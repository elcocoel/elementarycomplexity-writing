### Tell us more about your project and what you're building

Simple Page is an Ethereum-native publishing tool that makes it possible to create, update, and distribute permanent websites using markdown, ENS, and IPFS.

At the product level, Simple Page combines three things that usually live in separate worlds:

1. a browser-based writing and editing surface
2. permanent publishing through ENS + IPFS
3. open, community-runnable infrastructure rather than a closed hosted platform

The result is a publishing primitive for Ethereum. A user can fork a template, edit in the browser, publish to their ENS name, and keep an auditable, on-chain history of changes. For more advanced use cases, Simple Page also supports publishing arbitrary static sites through a CLI, running community indexer nodes, Safe multisig workflows, RSS generation, and embedded `web3://` forms for contract-native pages.

Simple Page is not a social network, newsletter platform, or hosted CMS. It is public publishing infrastructure built on Ethereum.

### Why does it qualify as a public good? Why is it impactful?

Simple Page qualifies as a public good because it strengthens public access to sovereign publishing infrastructure rather than locking creators, researchers, and builders into proprietary platforms.

Its impact comes from six properties:

1. **Open source codebase.** The implementation is public and forkable.
2. **Ethereum-native addressing.** Sites live on ENS names controlled by their publishers.
3. **Permanent, verifiable publishing.** Content is published to IPFS and every update produces an on-chain record.
4. **Community-runnable infrastructure.** Others can run indexer nodes instead of depending on a single company-operated backend.
5. **Multiple entry points for Ethereum users.** Writers, documentation teams, DAOs, solo builders, and developers can all use the same publishing substrate.
6. **Narrative leverage for Ethereum itself.** The easier it is to publish durable Ethereum-native websites, docs, explainers, archives, and research, the stronger Ethereum's own knowledge layer becomes.

Simple Page is impactful because it does not only help one project publish. It helps make Ethereum legible. It gives the ecosystem a native medium for documentation, public writing, educational content, lightweight product sites, and governance-facing communication.

### What will this funding unlock for your project

This funding would unlock three things at once: **traction, infrastructure resilience, and Ethereum-native content activation**.

First, it would help harden the infrastructure layer. Simple Page is not just a frontend; it depends on indexing, routing, and publish/retrieval flows that need to be dependable if the product is going to function as real public infrastructure. Funding here would improve the reliability of the network surface and reduce fragility around publishing and access.

Second, it would accelerate traction and adoption. The product already has real features, but getting more Ethereum users to understand why it matters requires tighter onboarding, stronger examples, and more distribution. Funding would help close the gap between technical capability and ecosystem adoption.

Third, it would support a content and template layer that benefits Ethereum directly. Simple Page can be more than a tool people happen to discover. It can become a publishing substrate for Ethereum documentation, explainers, research archives, lightweight protocol sites, DAO pages, and creator-owned educational media. Funding would make it possible to invest not only in the infrastructure itself, but in canonical examples, educational content, template packs, and outreach that help more people publish useful Ethereum-native content.

In practical terms, this means:

- improving infrastructure and operational durability
- improving the onboarding and publishing experience
- investing in templates, examples, and educational material
- supporting content and distribution work that helps Ethereum become easier to understand and easier to publish about

At a **$40,000** ask, the budget logic is:

- **$18,000** for infrastructure reliability, node resilience, indexing, and publish/retrieval hardening
- **$10,000** for onboarding, UX, templates, and documentation improvements
- **$7,000** for Ethereum-native educational content, canonical example sites, and tutorial material
- **$5,000** for community support, distribution, and ecosystem-facing activation

This is a compounding grant, not just an operating subsidy. Better infrastructure improves reliability. Better distribution improves traction. Better educational and publishing outputs increase the amount of Ethereum-native public knowledge that exists on top of the infrastructure.
